🔑 Platform Connections

Each platform: click the button → authorize → done permanently.

LinkedIn 1 click → automated forever
Why it needs a click: LinkedIn requires the account owner to approve the connection once — they call this "user consent" and enforce it for every API client.

What happens after: Our system gets a refresh token that lasts 365 days and auto-refreshes itself every 45 days. This click never repeats.
Instagram + Facebook 1 click → token never expires
Why it needs a click: Same as LinkedIn — Meta requires owner approval once.

What's different: Meta Page Tokens derived from a long-lived user token do not expire. One click today = working forever. No refresh ever needed.
Reddit fully automated via password grant
Reddit is different: Reddit's API supports "password grant" for script apps — meaning it uses your username + password + client credentials directly with no browser redirect.

What's needed: Create a Reddit app once at reddit.com/prefs/apps (script type), then paste the client_id and secret below. After that our system handles everything including token refresh.
Google / YouTube service account — bypasses consent screen
The problem: Publishing the OAuth consent app requires human verification at Google.

The bypass: Use a Service Account instead. Service accounts are for server-to-server authentication — they don't go through the consent screen at all. Tokens never expire (JWT-based).

Steps (done once):
1. console.cloud.google.com → IAM → Service Accounts → Create
2. Download the JSON key file
3. In YouTube Studio → Settings → Permissions → add service account email
4. Paste the JSON key below → saved to GitHub Secrets → done permanently

This one is genuinely a UI task. No workaround exists that isn't rebuilding Shopify Payments from scratch.
